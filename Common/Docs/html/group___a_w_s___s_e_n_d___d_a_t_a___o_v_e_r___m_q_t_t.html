Detailed Description

This example shows how to use the MQTT protocol to transfer the sensor data to an AWS server using Amazon FreeRTOS.

You need to have the below 4 valid files for the first time in SD card

  • rootCA.der - Root CA certificate in DER format. A valid file at the time of release is provided in the source/config folder.
  • xdk-cert.der - XDK certificate in DER format. A place holder is provided in the source/config folder which the user needs to update.
  • xdk-priv.der - Root certificate in DER format. A place holder is provided in the source/config folder which the user needs to update.
  • config.txt - Consists of the WLAN, SNTP and MQTT configurations in Json format. A place holder is provided in the source/config folder which the user needs to customize. These 4 files will be copied to WiFi file system and after that, only when the user wants to change this, they need to provide the files again. Please note that these files will be renamed as *.bkp after the successful copy. You can add your WLAN-Credentials in AppController.h
    which would be used if APP_USE_CONFIG_FILE is made "false". Several application related configurations (Like the MQTT Topic, etc) are also available in AppController.h
    . The software will fetch the SNTP time and set the same in the WiFi chip. It will subscribe to a topic and publish the sensor datas periodically (APP_MQTT_DATA_PUBLISH_PERIODICITY) to the same topic. When running the program if you keep the USB plugged in to the PC, you can see in the console output of the XDK-Workbench the communication status.

static

Currently, upon abrupt disconnection the event from the WiFi chip is not propagated / notified to the application. Until then, we manually validate prior to ever HTTP Rest Client POST/GET cycle to make sure that the possibility of software break is reduced. We check if the IP is valid to validate the same.

If there is no connectivity we will restart the node after 10 seconds.
