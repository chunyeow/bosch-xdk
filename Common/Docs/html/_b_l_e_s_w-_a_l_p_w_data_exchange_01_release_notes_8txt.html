* Profile changes:
[MOD] The ALPWISE DATA Exchange SERVER Rx Characteristic is now allowed to receive Write Without Response Commands. It allows a CLIENT to be able to send data in an non reliable way.
[MOD] API Change : BLEALPWDATAEXCHANGE_CLIENT_SendData now takes an extra parameter to provide type of packet (reliable/not reliable) used when sending this data packet. If you want to keep the same behavior as before specify BLEALPWDATAEXCHANGE_CLIENT_DATA_RELIABLE as parameter.
The non Reliable packets allow a better bit rate
